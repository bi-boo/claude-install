Section 03
安装与启动
1
运行安装脚本
耗时约 3–10 分钟，全程自动
  1. ①
    右键点击 02右键打开安装.command → 选「打开」
    必须用右键→打开，不能直接双击。这是 macOS 对外部脚本的安全机制，首次运行必须这样操作，之后才能直接双击。
    ⚠️ 首次必须右键打开
  2. ②
    弹出黑色终端窗口后，脚本自动完成以下步骤
    • 安装 Git（若弹出安装窗口，点「安装」，装完后按回车继续）
    • 安装 Node.js
    • 安装 Claude Code
    • 写入 API 配置（自动完成，无需手动填写）
  3. ③
    中途会提示输入 Mac 开机密码
    终端显示 Password: 时，输入你的 Mac 开机密码（不是 Apple ID）。输入时屏幕不会显示任何字符，这是正常的，输完直接按回车。
  4. ④
    出现「安装成功」提示即完成
    看到成功提示后，关闭这个终端窗口，继续下一步。
2
开始使用
安装完成后，按以下顺序操作
  1. ①
    关闭刚才的终端窗口
    安装脚本运行的终端要关掉，必须开新终端才能让配置生效。
  2. ②
    重新打开「终端」应用
    Spotlight 搜索「终端」，或 Finder → 应用程序 → 实用工具 → 终端。
  3. ③
    输入 claude 按回车启动
  4. ④
    遇到权限请求，点「允许」
    macOS 可能弹出「终端想要访问XXX文件夹」的系统弹窗，点「允许」即可，这是正常的。
Section 04
操作授权说明

安装脚本已经为你预置了一批常用操作的授权——比如读写文件、运行 Python/Node 脚本、管理文件等，这些你不需要每次手动同意。但仍有一些不常见的操作没有预置，遇到时 Claude Code 会弹出询问，你会看到三个选项：

选项 含义 建议
Allow 允许这一次操作，下次同类操作还会再问 刚开始用时推荐选这个，每次都看看它要做什么
Allow always 允许这次，并且以后类似操作不再询问 熟悉后，对信任的操作选这个，省掉重复确认
Deny 拒绝这次操作 不确定某个操作时随时可以拒绝
关于工具安装
使用过程中 Claude Code 可能会自动安装一些执行代码所需的工具包（如处理 Excel 需要的 Python 库等），遇到这类请求选择「允许」即可，这是正常工作流程。

Section 06
常用指令

进入对话后，直接在输入框里输入斜杠指令。按使用频率分三组：

每天都用
/clear
清空对话重新开始
清掉当前全部对话内容，完全重头开始。话题切换、对话跑偏时用。
每天都用
/resume
继续上次对话
弹出历史对话列表，选一条接着上次继续。关掉终端重开后找回之前进度用这个。
每天都用
/compact
压缩但保留上下文
对话太长 Claude 开始「记性变差」时用。和 /clear 的区别：会保留一份摘要，不会完全丢失背景。
项目配置
/init
为项目创建指令文件
让 Claude 扫描当前文件夹并自动生成一份 CLAUDE.md，写清楚这个项目是什么、有什么规则。之后每次进这个文件夹都会自动读取。
项目配置
/config
打开设置面板
可视化配置界面，能查看和修改 AI 行为偏好，包括是否开启自动压缩、主题颜色等。
项目配置
/memory
查看长期记忆
Claude 跨会话记住的信息都在这里，比如你的职位、项目背景、偏好设置。可以在这里查看和管理。
参考
/help
查看所有指令
显示当前版本全部可用指令的完整列表，遇到不熟悉的功能时随时查。
Section 07
如何配置

Claude Code 没有图形界面的"设置"页面。所有配置都通过文件来实现——你在文件里写下规则和偏好，Claude 每次启动都会读取并遵守。

配置文件存放在哪里

安装完成后，Claude Code 会在你的电脑上自动创建一个配置文件夹，路径是：

/Users/你的用户名/.claude/
这个文件夹是隐藏的

macOS 默认隐藏以点号开头的文件夹（如 .claude），在 Finder 里直接看不到。按以下快捷键可以随时切换显示/隐藏：

⌘ Command + ⇧ Shift + . 再按一次恢复隐藏
如何确认你的用户名

公司 IT 配置的 Mac 用户名通常是英文，不一定是你自己的名字。两种方式确认：

① 打开终端，输入 whoami 回车，显示的就是你的用户名 ② 打开 Finder，点左侧 🏠 图标的文件夹，看窗口顶部的文件夹名称
目录结构
~/.claude/
├── CLAUDE.md     ← 全局指令
├── settings.json  ← 技术设置
├── memory/       ← 长期记忆
└── rules/        ← 规则文件
你的工作文件夹/
└── CLAUDE.md     ← 项目指令
~/.claude/CLAUDE.md
全局指令 · 最重要
对你打开的所有项目都生效。适合放"不管干什么都适用"的偏好，如：所有回答用中文、解释技术内容用通俗语言、修改文件前先告诉我要改什么。安装脚本已预置了一份。
工作文件夹/CLAUDE.md
项目指令 · 按需设置
只对当前这个文件夹生效。适合放项目专属背景，如：这个文件夹是整理用户反馈的、不要动「已归档」目录。打开项目时，Claude 会同时读取全局指令和项目指令。
~/.claude/settings.json
技术设置 · 不需要手动改
存放 API 地址、密钥等技术参数，安装脚本已自动写好，一般不需要动。
~/.claude/memory/
长期记忆 · 自动维护
Claude 在对话中学到的值得跨会话记住的信息会自动存在这里，下次新会话也能记得，比如你的职位、常用的工作背景。
如何修改全局指令
启动 Claude Code 后直接说：「帮我打开全局配置文件，我想加几条指令」，或者「我是产品经理，帮我在全局配置里加上：所有输出用中文、解释技术内容时用通俗语言、修改文件前先列出改动计划」——它会直接帮你写进去。
Section 08
遇到问题？
双击 .command 文件没有反应
  • 右键点击文件
  • 选「打开」（不是双击）
弹窗提示「无法验证开发者」
  • 点「取消」关掉弹窗
  • 打开「系统设置 → 隐私与安全性」
  • 找到相关提示，点「仍要打开」
安装中途卡住不动了
等待 5 分钟，若仍无反应，关闭终端后重新右键运行安装脚本。已完成的步骤会自动跳过，不需要从头开始。
输入 claude 后没有反应
确认已关闭旧终端窗口，重新打开一个新的终端再试。安装完成后必须用新终端才能识别到 claude 命令。
